------------------------------------------------------------
CRITICAL SYNTAX RULES
------------------------------------------------------------

1. Opening brace MUST appear on the next line.

VALID:
function main()
{
    ...
}

INVALID:

2. Increment operators are NOT supported.
   - i++
   - ++i
   - i--
   - --i

3. Compound assignment is NOT supported.
   - +=
   - -=
   - *=
   - /=

4. Loop increments MUST use explicit assignment:

VALID:
i = i + 1

INVALID:
i++
i += 1

5. Do NOT generate Java/C-style syntax.

6. All variables must be declared before use.

7. No unsupported operators may be used.

------------------------------------------------------------
HAWK LANGUAGE GRAMMAR (EBNF)
------------------------------------------------------------

program         ::= function_def+

function_def    ::= "function" identifier "(" parameter_list? ")" newline block

parameter_list  ::= parameter ("," parameter)*
parameter       ::= "var" identifier | identifier

block           ::= "{" statement* "}"

statement       ::= var_decl
                  | array_decl
                  | assignment
                  | exec_stmt
                  | return_stmt
                  | if_stmt
                  | else_stmt
                  | for_stmt
                  | while_stmt
                  | do_while_stmt
                  | expr_stmt

var_decl        ::= "var" identifier "=" expression
array_decl      ::= "var" identifier "[]" "=" "new" "var" "[]"

assignment      ::= identifier "=" expression
exec_stmt       ::= "exec" expression
return_stmt     ::= "return" expression

if_stmt         ::= "if" "(" condition ")" block
else_stmt       ::= "else" block

for_stmt        ::= "for" "(" var_decl_or_assignment ";" condition ";" assignment ")" block

var_decl_or_assignment ::= var_decl | assignment

while_stmt      ::= "while" "(" condition ")" block
do_while_stmt   ::= "do" block "while" "(" condition ")"

expr_stmt       ::= expression

condition       ::= expression comparator expression

comparator      ::= "==" | "!=" | ">" | "<" | ">=" | "<="

expression      ::= literal
                  | identifier
                  | identifier "(" argument_list? ")"
                  | identifier "[" expression "]"
                  | identifier "->" identifier
                  | expression operator expression
                  | "`exec" expression "`"

operator        ::= "+" | "-" | "*" | "/" | "%"

argument_list   ::= expression ("," expression)*

literal         ::= number | string

identifier      ::= letter (letter | digit | "_")*
number          ::= digit+
string          ::= "\"" (any_char)* "\""

------------------------------------------------------------
STRICT SYNTAX & FORMATTING CONSTRAINTS
------------------------------------------------------------

1. The opening brace "{" of a function MUST appear on the next line.

   Valid:
   function main()
   {

   Invalid:
   function main() {

2. The opening brace of if, for, while, and do blocks MUST appear on the next line.

3. Increment and compound operators are NOT allowed:
   - i++
   - ++i
   - i--
   - i += 1
   - i -= 1
   - i *= 2
   - i /= 2

   Only explicit assignment is allowed:
   i = i + 1

4. The following shorthand expressions are NOT allowed:
   - +=
   - -=
   - *=
   - /=
   - ++
   - --

5. Every statement MUST be on its own line.

6. The generated code MUST follow the formatting style shown in the examples.
   Do NOT use C-style formatting shortcuts.
